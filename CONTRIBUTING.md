# Contributing Guidelines

Thank you for your interest in contributing to this repository.

This project is a **curated index of Computer Science learning resources**.
Quality, clarity, and long-term value are the main criteria for acceptance.

---

## What You Can Contribute

You may contribute by:
- both well-known and lesser-known resources are welcome, as long as they are clear, correct, and genuinely useful.
- suggesting quality learning resources
- improving descriptions or organization
- fixing mistakes or broken links
- proposing structural improvements

Low-quality or clearly redundant content is unlikely to be accepted.

---

## How to Contribute

### 1. Open an Issue
Use an issue to:
- suggest a new resource
- report an error or broken link
- discuss structural changes

Clearly explain **why** the contribution is valuable.

### 2. Open a Pull Request
- Fork the repository
- Create a new branch
- Follow the existing structure and formatting
- Keep changes focused and minimal
- Open a Pull Request with a clear description

---

## Formatting Rules

- Keep descriptions concise and factual
- Avoid marketing language
- Prefer authoritative and widely trusted sources (Even lesser-known repositories are fine if they are well done.)
- Follow the existing section templates exactly

---

## Review Process

Acceptance is based on:
- relevance
- educational value
- consistency with repository goals
